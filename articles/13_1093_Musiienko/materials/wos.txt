# Web of Science Search Strategy for Scoping Review

## Generative AI and the Transformation of Design Education

---

## I. Search Strategy Overview

### Rationale

For a PRISMA-ScR compliant scoping review, you need a **reproducible, comprehensive, and documented** search strategy. The queries below are organized by:

1. **Core intersection** (AI + Design Education)
2. **Disciplinary specificity** (individual design fields)
3. **Pedagogical themes** (assessment, curriculum, studio pedagogy)
4. **Theoretical constructs** (creativity, collaboration, skill)

### Search Parameters

| Parameter | Recommendation |
|-----------|----------------|
| **Database** | Web of Science Core Collection |
| **Date range** | 2017–2025 (captures pre-GPT and post-GPT eras; allows periodization) |
| **Document types** | Articles, Reviews, Early Access |
| **Language** | English (or all languages with English abstract) |
| **Search fields** | Topic (TS=) which searches Title, Abstract, Author Keywords, Keywords Plus |

## VIII. Search Execution Protocol

### Step 1: Run Individual Queries

Execute each query separately and document:
- Date of search
- Number of results
- Any modifications made

### Step 2: Combine and Deduplicate

```

#11 OR #12 OR #13 OR #14 OR #15 OR #16 OR #17 OR #18 OR #19
```

### Step 3: Apply Filters

| Filter | Setting |
|--------|---------|
| Document Types | Article, Review, Early Access |
| Languages | English |
| Date Range | 2017–2025 |
| WoS Categories | Consider filtering by Education, Computer Science, Design, Arts, Architecture (optional—may lose relevant interdisciplinary work) |

### Step 4: Export and Screen

Export to reference manager (EndNote, Zotero, Mendeley) for:
1. Title/abstract screening
2. Full-text screening
3. Data extraction

## XI. Query Refinement Notes

### If Yield is Too High

- Add proximity operators: `"design education" NEAR/5 "artificial intelligence"`
- Restrict to Title only: `TI=` instead of `TS=`
- Narrow date range to 2020–2025
- Add Web of Science Category filters

### If Yield is Too Low

- Broaden terms (e.g., add "technology" alongside "AI")
- Use truncation more liberally: `design*`, `creat*`, `art*`
- Remove discipline restrictions
- Extend date range to 2015–2025

### High-Priority Queries for Pilot Testing

Start with Queries **1, 9, 13, and 14** as these are most central to your paper's focus and will help calibrate your screening criteria.
