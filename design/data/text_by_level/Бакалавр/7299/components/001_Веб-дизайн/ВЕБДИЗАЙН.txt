АНОТАЦІЯ КУРСУ
Курс покликаний ознайомити з основами сучасних вебтехнологій та вебдизайну. На
прикладі розробки вебсайту студенти мають змогу опанувати всі етапи створення
вебсайту: від пошуку ідеї, визначення структури та підготовки контенту до
проєктування інтерактивного прототипу у Figma, реалізації сайту у коді (HTML / CSS /
JavaScript) та фінальної публікації онлайн.
МЕТА ТА ЦІЛІ КУРСУ
Мета курсу: формування системи теоретичних знань з основ вебдизайну та
вебтехнологій, а також набуття практичних навичок зі створення вебсайтів.
Цілі курсу: надати студентам уявлення про суть, проблематику, теоретичні та
практичні підходи до вебдизайну, сформувати знання про основні концепції, методи
та засоби вебдизайну, а також отримати практичні навички з проєктування, створення
та публікації вебсайтів із застосуванням Figma, HTML, CSS, JavaScript, Git та інших
інструментів вебдизайну.
РЕЗУЛЬТАТИ НАВЧАННЯ
У разі успішного проходження курсу студент повинен:
● опанувати основи HTML і CSS та вміти створювати базові вебсторінки за
допомогою цих технологій;
● вміти використовувати JavaScript для додавання інтерактивності та
динамічного вмісту вебсторінок;
● ознайомитися з принципами адаптивного дизайну для створення адаптивних
вебсайтів;
● вміти розробляти та створювати орієнтовані на користувачів вебсайти із
врахуванням вимог стандартів вебдоступності;
● розуміти принципи дизайну взаємодії з користувачем (UX) і вміти створювати
вайфрейми та прототипи для вебсайтів;
● отримати знання про актуальні тенденції вебдизайну, найкращі практики та
етику та вміти застосовувати їх;
● вміти працювати самостійно та як частина команди над проєктом з вебдизайну;

● вміти презентувати проєкти вебдизайну, а також мати досвід отримання
відгуків та критики;
● мати чітке розуміння принципів, методів та інструментів вебдизайну та мати
можливість використовувати ці знання для продовження навчання та розвитку
як вебдизайнера.

ЗАВДАННЯ, ВИМОГИ ДО ЇХ ВИКОНАННЯ ТА КРИТЕРІЇ ОЦІНЮВАННЯ
Завдання:
Для успішного складання заліку студент повинен у повному обсязі виконати наступні
завдання:
1. Створити персональну вебсторінку, використовуючи виключно HTML.
Опублікувати сторінку онлайн за допомогою сервісу GitHub Pages.
2. Запроєктувати інтерактивний прототип вебсайту у Figma з використанням
Autolayout, Components, Variants.

3. Розробити фінальний вебсайт у коді (HTML, CSS, JS). Опублікувати онлайн за
допомогою сервісу GitHub Pages.
Вимоги до фінального вебсайту
● односторінковий вебсайт “персональне портфоліо” або на іншу, погоджену з
викладачем тему;
● мінімум три повноекранні секції. Наприклад: Головна, Про мене, Роботи,
Контакти тощо;
● фіксоване навігаційне меню (з ефектом smooth scroll);
● верстка із застосуванням CSS Flexbox;
● застосування Google Fonts;
● авторський контент;
● оптимізація зображень під веб;
● застосування JavaScript компонента галереї LightGallery.js
● Favicon.ico, Open Graph metatags;
● публікація онлайн з використанням сервісу Github Pages на унікальній адресі
сайту <user>.github.io.

що перевищує дозволений ліміт буде знижено підсумковий бал. Зниження
підсумкової оцінки може сягати до 25 балів. Якщо впродовж семестру відсутність
перевищила присутність, то атестація студента є під загрозою. Обов’язком студента є
повідомити викладача про причини відсутності та надати супровідну документацію.

здійснюється відповідно до положення «Про порядок ліквідації академічних
заборгованостей».

Зверніть увагу: цей силабус є загальним керівництвом для курсу. Викладач
залишає за собою право вносити зміни до навчального плану за необхідності
протягом семестру. Студенти несуть відповідальність за те, щоб бути в курсі
будь-яких змін або оновлень. У випадку виникнення запитань чи сумнівів,
зверніться до свого викладача.
